# Project Codes

Every ticket in Tentacles belongs to a project code. The code appears in the Ticket's Serialized ID (e.g., `AC-OPS-001`) and tells you instantly which area of work the ticket belongs to.

---

## How Codes Work

Project codes are stored as a select enum on the Tickets database. The onboarding agent sets them up during initial configuration by adding the values to the enum via the Notion MCP. From there, every ticket must have a valid project code.

Codes follow a simple pattern:
- **Internal codes:** `{PREFIX}-{SUFFIX}` — where PREFIX is your company's 2-3 letter abbreviation
- **Client codes:** typically 3-4 letters from the client name

Ticket IDs are auto-generated by a Notion formula: `<Project Code>-<zero-padded number>`. For example, if you have the code `AC-OPS` and it's the 7th ticket in that code, the ID is `AC-OPS-007`.

---

## Standard Internal Suffixes

The onboarding agent proposes these standard suffixes. You can add, remove, or rename any of them.

| Suffix | Full Code (example) | Purpose |
|--------|---------------------|---------|
| OPS | AC-OPS | Operations & admin — day-to-day ops tasks, internal processes, vendor management |
| WEB | AC-WEB | Website & web projects — frontend, CMS updates, web app work |
| SRV | AC-SRV | Servers & infrastructure — hosting, DevOps, databases, cloud infrastructure |
| ONB | AC-ONB | Team onboarding — new hire setup, documentation, access provisioning |
| TKT | AC-TKT | Ticket system maintenance — improvements to the Tentacles system itself |
| OA | AC-OA | Legal & operating agreements — contracts, NDAs, operating docs |
| TAX | AC-TAX | Tax & finance — tax filings, accounting, financial admin |
| SITE | AC-SITE | Public site & marketing — landing pages, SEO, content marketing |
| DATA | AC-DATA | Data & analytics — reporting, dashboards, data pipelines |
| AGENT | AC-AGENT | AI agent work — building, testing, and deploying AI tools |

---

## Client Codes

Client codes use the client name rather than a suffix. The onboarding agent suggests a 3-4 letter code for each client.

Examples:
- Persimmon Capital → `PERS`
- Nova Industries → `NOVA`
- Riverstone Analytics → `RVST`

A client code ticket would look like `PERS-007` — the 7th ticket related to Persimmon Capital.

---

## Adding Codes Later

You can add new project codes at any time:

- **Ask the agent:** "Add a new project code AC-TRAIN for training and workshops" — the agent updates the Tickets database enum via MCP and confirms.
- **Add a new client code:** "Add a client code RVST for Riverstone Analytics" — same process.

Never add codes manually via the Notion UI if you want the agent to use them reliably. The agent validates against the config's code list, so if you add codes manually, ask the agent to sync the config afterward.
